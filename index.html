個人簡介

Wehlep的指導教練們好，我是Steven，感謝撥冗閱讀個人網頁。
目前33歲，出社會以來的工作經歷分成2個階段。
個人退伍後，充分利用自己大學所學，順利進入國際貿易業務相關職位就職。在職期間主要對應美國與日本的客戶，進行產品的專案開發。
國外業務職位的工作經驗約4年，當時對於國外業務相關的薪水不是很滿意、內部也沒有往上爬的機會，且也想嘗試新的領域的事物，不過沒有特別的想法，所以2018年決定去日本飯店業工作，打算邊做邊思考職涯發展。
經過線上求職之下順利進入一間日本大型連鎖飯店－大江戶溫泉物語 工作，職務內容主要在飯店餐廳接待客人入場，用餐點酒說明及桌邊服務。
在日本工作時，有個當兵同梯的朋友跟我推薦有空可以學一些跟前端設計有關的東西，如果有興趣的話未來或許可以試著到科技業工作。
那時候對於程式語言的東西很陌生，所以去w3school看看這些是做在什麼用的，大概只有用放假空閒時間看一些學一點簡單的操作，但仍不是學習的主要重心，當下只覺得JavaScript很有趣。在飯店工作約2年後，疫情的爆發影響日本觀光業，飯店業也開始縮編人員，約聘人員們都是被資遣的首要對象之一，所以當時不得不回台灣生活。
在2021年回到台灣後，審視一下自己的狀況，對於程式語言有興趣及熱枕，html、css與JavaScript已經有初步的認識及基礎，也有足夠的存款，下定決心拚轉職科技業。

為了成為軟體工程師，曾做過什麼努力？若有具體專案作品請分享給我們。

w3Schools

w3Schools是我學前端最初接觸的一個學習平台。資料庫幾乎是涵蓋所有HTML、CSS、JS和Python等的語法。
除了每種語法的說明外，還有簡易題目可以操作，獲取個人的分數，在初期常常忘了語法怎麼使用時，就會到w3school或MDN查詢，相當實用的工具書。

學習狀況如下:

  • HTML: 54%
  • CSS: 59%
  • JavaScript: 44%
  • Point: 497

Udemy

在接觸前端設計之前，試著到PTT爬文參考一些自學前端的前輩們的文章，都推薦可以去Udemy買課程，因為在特價期間優惠的價格只要幾百塊，很適合初學者們試水溫。

購買的課程有:

  • The Complete Web Developer in 2023: Zero to Mastery
  • 2022網頁開發全攻略(HTML, CSS, JavaScript, React, SQL, Node, more)

布魯斯前端

除了使用Udemy平台學習以外，也有到YouTube觀看布魯斯前端的【前端速成 JavaScript JS 快速入門】影片，布魯斯講的內容是JS基礎觀念，介紹JS的核心語法後有影片可以跟著實作一個備忘錄小App

六角學院

在看完Udemy的課程後，因為都是英文講解，怕漏掉重要基礎觀念，所以有考慮參加較大型課程的平台，才加入了六角學院。
六角學院在每個章節都講得非常詳細，且也有定期需繳交的作業，由助教做批改回饋訊息，比之前學習的平台上內容更為完整。

購買的課程如下:

  • 使用 HTML、CSS 開發一個網站
  • JavaScript 必修篇 - 前端修練全攻略
  • JavaScript 核心篇
  • 一變應萬變的響應式網頁設計
  • 使用 jQuery 打造互動性網頁動畫效果
  • Bootstrap 5 網頁切版整合術
  • Vue 3 實戰影音課程
  • 使用 jQuery 打造互動性網頁動畫效果

Alphacamp

雖上完六角學院的課程，但是對於專案製作上還是會常常遇到撞牆的時期，JS的語法使用、網頁的排版等，使用上總是覺得自己仍對前端技術不是很熟悉。
後來無意間看到AC有優惠，就購買學期一的內容試讀，試上之後發現他們的課程內容非常完整，有影片及文字的交互呈現下，讓學習者可以更沉浸在學習前端技術。
AC與六角一樣有助教批改作業機制，只是AC的作業比六角的來的更多且程度稍微在難一些些，此外線上直播實作的課程也很實用。

購買的課程如下:

  • 學期 1：程式設計入門: Completed
  • 學期 2-1：JavaScript 前端開發: Completed
  • 學期 2-2：軟體開發實務入門: Completed
  • 學期 2-3：前端開發實務(Cohort Start Date：2023-03-13) : 60%
  • 學期 3：軟體工程師養成 - 前端課程(Cohort Start Date：2023-04-24)

paiza

曾經在網路上看到有一款遊戲是跟程式語言有關的遊戲(日式王道RPG)エンジニア騎士とクエリの魔女。它是一款可藉由解題破關獲得裝備，看了覺得很有興趣；經查詢後，才知道是paiza這家公司所製作的遊戲，同時它也是一個程式語言的學習平台，大多的初階課程都是免費觀看。
除了學習課程以外，他們還有提供學習者與IT公司媒合的服務，他們會看你在他們平台上的使用表現再決定是否將你的履歷主動推薦給IT公司。
後期我比較常用的是スキルチェック這項服務，它類似Leetcode提供學習者解題，與Leetcode稍微不一樣的是嚴禁使用者將解題的code內容PO在網路上， 所以在網路上比較難看見其他人的解題分享，為的是維護他們替IT公司在尋找或媒合人才時，可以參考學習者拿到的分數且比較有可信度。

是否有想要加入的軟體公司？為什麼想加入該公司？

如果有想要加入的軟體公司的話，絕對是Ubiquiti這家美商。Ubiquiti在台灣深耕10年多了，產品以wifi等網路通訊設備起家。
雖然嚴格來說是一家硬體公司，但隨著產品逐漸多元化，他們從主力網通產品，逐步跨足EoT裝置、監控設備方案以外，連軟體也是他們現階段的核心開發重點，期望透過軟硬整合，讓客戶跟他們購買的設備都是一整套完整的Ubiquiti產品。
在我從事國外業務時，有幸與Ubiquiti的團隊一起合作共同開發一款小型的手組式Rack機架，雖然與軟體的部分無關，但在與機構設計團隊合作的過程中了解到他們公司的專業能力與團員們的自我要求。
像是在產品驗貨，他們對於產品的尺寸公差及品質要求都有嚴格規定細節，如果做出來的產品達不到他們期望的話會請我們檢討製程，確定是不是可以達到要求? 不行的話為什麼? 解決方案有哪幾種?
開發期間雖然常常遇到撞牆期，且對於板金加工的知識不是非常有經驗的我，必須要一一去詢問各個單位的前輩，一一突破Ubiquiti給的難題。經過一年半的磨合下，順利導入量產並開始在世界各地販售。
公司內部文化除了彈性自主以外，更善用公司內現有的員工才華。只要是任職滿一年的員工，都可以申請轉調到自己想去的部門。
我自己很嚮往他們公司的內部文化和工作的方式，與自己做事的態度較為相近，都是喜歡尋找問題和解決問題，如果自己未來在轉職成功後的階段中，絕對會嘗試挑戰進入該家公司。

請描述一件讓你產生明顯負面情緒的事情，你如何處理該情緒？

在退伍成為新鮮社會人後，進去一家小型公司上班，職位是國外業務銷售員，主要是開發國外潛在客戶購買公司的CNC銑刀產品。
當時進去的新人都不會被分配到既有的客戶，必須從0開始培養出自己的客戶群，怎麼開發尋找客戶就成了新人們的課題。
那時候請教前輩們開發方式，他們說就是用Google去搜尋關鍵字，找到相關的廠商後發Email給他們做個簡單的產品介紹，客戶如果有興趣的話就會跟你聯繫詢價。剛開始都是照這樣的模式去開發客戶，但經過了3個月後都沒有任何成效，發出去的Email有100封可能0封都沒消息，如果有回覆的往往都會拿大陸的便宜價格比較，然後就不了了之。
在那樣的環境下會感到非常沮喪且無力，負面情緒也很重，腦子只想著換工作。後來與朋友家人聊過之後，其實不管到哪裡一定都會碰到類似的挑戰，如果在這邊就放棄了，換去新的地方你一定又會很快選擇投降。
與他們談完之後，覺得不應該這麼簡單就輕言放棄。從那之後大概花了一個禮拜的時間思考，到底要怎麼樣才能將Google的搜尋達到最大化，並找到那個我想要的客戶，找到後可以吸引他的目光?
研究之下Google其實有個進階搜尋的功能，可以鎖定地區跟網域，配上關鍵字後再設定想開發地區的當地網頁資料，過濾後的這些網站就可能是我想找的客戶群，執行後果不期然列出的大多都是我想開發的經銷商。找到目標以外還不足以完全留住客戶，如果Email內容只是簡單的文字敘述，收到的經銷商們都會覺得是普通的垃圾郵件，因此，自己開始用PhotoImpact做一些漂亮的文宣海報，讓客戶開啟信件就可以抓住他們的目光，比起文字，看圖片更是直覺地呈現自己的產品，除此之外，也簡明扼要地說明來信重點。
在這樣的開發下，不到一個月，就有10家以上的客戶來詢價，除了一些價格上的關係沒有留住以外，成交的客戶至少有5家左右。對於社會新鮮人來說，這是非常有成就感的事情。

關於這份申請網頁，請分享一個你開發時的技術心得。

製作這份網頁前，我有想著要將自己到目前為止自學的程式語言技巧，呈現在這個網頁上。原本的構想是做成一般網站常見的排版，有navbar、hamburger menu等方式，可以一項一項點進去看，但考量到瀏覽者的觀看及操作方便性，不如單一一個畫面呈現還來得好，所以將構想草稿做了大改，改成一頁式瀏覽，並透過scroll的方式，增加一點小動畫。如果getBoundingClientRect().top小於瀏覽器window的innerHeight一定的比例的話，下一個section的動畫就會出現。
另外，針對第一道的題目200字以內的自我介紹，這邊我有想放上自己的履歷表，但礙於頁面的空間的關係，又另開一個分頁呈現自己的履歷。第3道題目在select選擇器上做了一些簡易的JS效果。使用者有點開再呈現即可，希望讓整個版面看起來更簡潔。
最後就是在排版上做一些調整，如文字的最大寬度、字體的間隙、行距高度，版面的色彩與對比等，希望面試官們閱讀的時候可以比較輕鬆。

其他想要對我們說的事情？

以上面試官們參考個人的學習狀況後，或許您會覺得我已經接受過很多課程的訓練，為什麼還想參加wehelp提供的培訓。
我自己的想法是因為現在年代在招募工程師的條件似乎又跟前幾年有點不太一樣，而現在前端的人才需求逐年下降，公司人資篩選新人的標準，都是希望能夠成為即戰力的一份子。所以我也希望自己的實力可以達到剛加入公司就能夠快速上線，成為工作戰力輸出的一個角色。
除此之外，在之前學習平台上課的過程中，個人欠缺多人合作專案的經驗，想藉由wehelp的上課方式，累積這方面的實力。未來在尋找工作的時候也比較好呈現自己為專案中所完成的部分。※目前多人合作只有Alphacamp的學期3有一個，除了這個以外就沒有了。
希望有這個機會可以參加Wehelp的培訓課程，謝謝。
